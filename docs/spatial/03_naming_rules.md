> **원본**: `1. Reference data/실내공간정보 구축 작업규정/[별표 5] 실내공간정보 레이어 명명규칙(실내공간정보 구축 작업규정).pdf` (2쪽)
> **작성일**: 2026-07-05
> **목적**: POC 공간 스키마(`app/src/domain/spatial`) 매핑을 위한 분석

## 1. 코드 3종 패턴 요약

| 코드 | 패턴 | 예시 (원문) |
|---|---|---|
| 가. 레이어코드 | `L_` + ①건물ID + `_` + ②층 + `_` + ③구분 + `_` + ④구조 | 판매 시설물 → `L_UFID코드_B01F01_SL_FC` |
| 나. 3차원객체코드 | `M_` + ①건물ID + ②층 + `_` + ③구분 + ④구조 + `_` + ⑤시설구분 + ⑥일련번호 | 지하1층 천정 → `M_UFID코드B01B01_ISCL_0000001` / 지하1층지상1층 계단 → `M_UFID코드B01F01_MVFC_0100001` |
| 다. 3차원텍스쳐코드 | `T_` + ①건물ID + ②층 + `_` + ③구분 + ④구조 + `_` + ⑤시설구분 + ⑥일련번호 + `_` + ⑦텍스처번호 | 지하1층 천정 → `L_UFID코드B01B01_ISCL_0000001_01` (원문 표기) |

> **원문 오탈자 주석**: 다. 텍스쳐코드의 규칙 정의는 접두 `T_`인데 예시 2건은 모두 `L_` 접두로 표기되어 있다(`L_UFID코드B01B01_ISCL_0000001_01`, `L_UFID코드B01F01_MVFC_0100001_02`). 규칙 정의를 기준으로 `T_` 접두가 올바른 것으로 판단한다.
>
> 3차원객체코드는 레이어코드와 달리 ①건물ID와 ②층 사이에 구분자 `_`가 없고, ③구분과 ④구조 사이에도 구분자가 없다(예: `M_...B01B01_ISCL_...`).

## 2. 구성 요소 입력값 (가·나·다 공통)

| 구분 | 입력값 |
|---|---|
| ① 건물 ID | UFID(공간객체등록번호) CHAR(17) |
| ② 층(FLOOR) | 지하1층 → `B01B01`, 지상1층 → `F01F01`, 로비층 → `L01L01`, 지하1층지상1층 → `B01F01` |
| ③ 구분(DIVISION) | 이동 → `MV`, 안내 → `IF`, 편의 → `CV`, 운수 → `TP`, 안전 → `SF`, 소방 → `FF`, 판매 → `SL`, 의료 → `MD`, 주거 → `HS`, 업무 → `BS`, 관리 → `MN`, 구조 → `IS` |
| ④ 구조(CLASSIFY) | 천정 → `CL`, 벽 → `WL`, 바닥 → `FL`, 문 → `DR`, 창문 → `WD`, 기둥 → `PL`, 시설물 → `FC` |
| ⑤ 시설구분 | 3차원 시설구분코드 참조 (§4) — CHAR(2) |
| ⑥ 일련번호(SERIAL) | CHAR(5) |
| ⑦ 텍스처번호 | CHAR(2) — 텍스쳐코드에만 사용 |

### 층(FLOOR) 코드 해석

CHAR(6) = 시작층(3자) + 끝층(3자). 접두 `B`(지하)/`F`(지상)/`L`(로비) + 2자리 층번호. 단일 층은 동일 코드를 반복하고(`F01F01`), 여러 층에 걸친 객체(계단 등)는 시작층-끝층을 결합한다(`B01F01`).

### DIVISION 코드와 별표 2 분류의 대응

| DIVISION | 중분류 | 대분류 | | DIVISION | 중분류 | 대분류 |
|---|---|---|---|---|---|---|
| `MV` | 이동 | 개방 | | `SL` | 판매 | 개방 |
| `IF` | 안내 | 개방 | | `MD` | 의료 | 개방 |
| `CV` | 편의 | 개방 | | `HS` | 주거 | 고유 |
| `TP` | 운수 | 개방 | | `BS` | 업무 | 고유 |
| `SF` | 안전 | 개방 | | `MN` | 관리 | 관리 |
| `FF` | 소방 | 개방 | | `IS` | 구조 | 관리 |

> **주의**: 별표 5의 CLASSIFY에는 `공간(RM)`이 등장하지 않는다(구조 6종 + 시설물 FC만 7종). 공간 `RM`은 별표 6 속성입력의 CLASSIFY 정의에서 추가된다 → [04_attribute_input.md](./04_attribute_input.md) §2 ⑦. 별표 6의 공간 레이어정의서 예시(`L_B00100000001AULW1_F01F01_MV_RM`)가 레이어코드에 RM을 사용함을 확인해 준다.

## 3. 실제 UFID 예시

별표 6 레이어정의서 예시에서 UFID mock 값 `B00100000001AULW1`(CHAR 17)이 사용된다. 완성 예:

- 공간 레이어코드: `L_B00100000001AULW1_F01F01_MV_RM` (지상1층 이동 공간)
- 경로 레이어코드: `L_B00100000001AULW1_F01F01_MV_LINE` (경로네트워크 — 별표 6 나. 참조)

|---|---|---|

## 6. POC 매핑 관련 메모

- 레이어코드는 `SpatialSpace.layerCode`의 형식, 3차원객체코드는 `SpatialFacility`의 id 형식으로 채택한다.
- 텍스쳐코드(`T_`)는 POC에 텍스처 개념이 없어 제외한다 → [06_schema_mapping.md](./06_schema_mapping.md) 참조.
