## 1. 품질기준 구조

품질요소(구분) 5종 × 품질검사항목 19종. 검사방법은 **표본(20%)검사**(무작위 추출)와 **실내(전수)검수** 두 가지이며, 적정성 기준은 오류율형(1% 이하)과 적합율형(90% 이상)으로 나뉜다.

## 3. 품질검사표 서식

2쪽의 품질검사표는 위 19개 항목을 1.완전성(가~바) / 2.논리일관성(가~마) / 3.위치정확성(가~라) / 4.주제정확성(가~다) / 5.기타(가) 체계로 나열하고, 항목별로 `품질검사 대상(개) / 기준이내(개) / 완성도(%) / 품질적정성(적, 부) / 비고`를 기입하는 서식이다. 하단에 품질적정성 판정자의 소속/직책/성명/서명란이 있다.

## 4. POC 매핑 관련 메모

POC는 구축 사업 검수가 아니므로 품질기준 전체를 구현하지 않는다. 다만 자동 검증 가능한 항목의 취지를 Phase 6 검증 규칙에 반영한다:

| 규정 항목 | POC 반영 |
|---|---|
| ID무결성 (별표5·6) | T2 naming 함수의 코드 생성/파싱 왕복 검증, T3 레지스트리의 id 유일성 |
| 분류코드 무결성 (별표5·6) | T2 코드 상수 테이블(DIVISION/CLASSIFY/시설구분/QAL·SLP·DIR·SKT·HCP) 기반 타입 제약 |
| 항목 분류·속성내용 정확성 (별표2·5·6) | validateGraph 규칙 8 — Scope/Asset 노드의 siteId/spaceId/assetIds 레지스트리 대조 (T6) |
| 단위공간정의·기하요소정의 일관성 (별표3) | 검증용 샘플 데이터의 footprint 볼록다각형·비중첩 설계 규칙 (T3), SVG 층 평면 뷰 육안 확인 (T7) |
| 데이터스키마 무결성(CityGML), 세밀도·가시화 일관성, 위치정확성 등 | 제외 — CityGML XML 산출물·텍스처·측량 좌표계가 POC 범위 밖 |
